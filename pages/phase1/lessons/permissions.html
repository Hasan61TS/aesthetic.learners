File Permissions & Ownership

Understanding Linux Permissions

Linux uses a simple but powerful system to control who can access files and what they can do with them.

r Read permission
w Write permission
x Execute permission

üí° Permission Analogy

Think of file permissions like a safe deposit box:
- Read (r) is like having permission to look inside
- Write (w) is like being able to put things in or take them out
- Execute (x) is like knowing the combination to open it

The chmod Command

The chmod command changes file permissions. It can use either symbolic (letters) or numeric (numbers) mode.

chmod +x file Add execute permission
chmod 755 file Set rwx for owner, rx for others
chmod -w file Remove write permission
Try Permissions

File Ownership

Every file in Linux has an owner and a group. The chown command changes ownership.

chown user file Change file owner
chgrp group file Change file group
ls -l List files with permissions

Practice Exercise

Try these commands:

  1. Create a file with touch
  2. Check its permissions with ls -l
  3. Make it executable with chmod +x
